# 後輩使用指南：如何用這個模板做你自己的問卷

## 你需要的東西
- 一個 GitHub 帳號（免費）
- 一個 Google 帳號（用來建試算表）

---

## Step 1：Fork repo

1. 進入學長姐給你的 GitHub repo 連結
2. 點右上角「**Fork**」
3. 點「**Create fork**」
4. 你現在有自己的副本了，之後所有操作都在你自己的副本上

Fork 完成後，你的問卷網址格式就已確定：
```
https://你的GitHub帳號.github.io/你的repo名稱/
```
先記下來，Step 2 會用到。

---

## Step 2：建立你自己的 Google 試算表 + 部署接收腳本

1. 建立新的 Google 試算表（空白即可）
2. 點「擴充功能」→「Apps Script」
3. 把 `apps-script.gs` 的內容全部貼上
4. 找到程式碼**最頂部**這一行，把你的問卷網址填進去：
```javascript
var SURVEY_URL = 'https://你的GitHub帳號.github.io/你的repo名稱';
```
5. 點「部署」→「新增部署作業」
6. 類型選「**Web 應用程式**」
7. 執行身分：「我」；存取權限：「**所有人**」
8. 點「部署」→ 複製產生的 URL，長得像這樣：
```
https://script.google.com/macros/s/AKfycb.../exec
```

**把這串 URL 複製起來，Step 3 會用到。**

---

## Step 3：用 AI 幫你產出正確的 config.json

config.json 是這個模板的核心，裡面包含你的研究架構、情境文字、量表題目等設定。
**建議直接用 AI（ChatGPT、Claude 等）來幫你修改，比自己手動改更不容易出錯。**

### 3-1：取得目前的 config.json 內容

1. 在你 fork 後的 repo 頁面，點 `config.json`
2. 點右上角「**Raw**」按鈕
3. 全選（Ctrl+A / Cmd+A）→ 複製（Ctrl+C / Cmd+C）

### 3-2：取得 ai_guide.md 的內容

1. 回到 repo 頁面，點 `ai_guide.md`
2. 同樣點「**Raw**」→ 全選 → 複製

### 3-3：把兩份內容丟給 AI

打開任一 AI 工具（ChatGPT、Claude 等），貼上以下格式：

```
以下是我的問卷模板設定指南：

【ai_guide.md 的內容貼在這裡】

以下是我目前的 config.json：

【config.json 的內容貼在這裡】

請依照指南，逐步問我研究架構、情境文字和量表題目，幫我產出新的 config.json。
```

AI 會開始問你問題，例如：
- 你的研究有幾個自變項？
- 每個自變項的高低情境文字是什麼？
- 你的量表有哪些題目？
- 每個組合預計收幾份問卷？（配額控制）

**照實回答即可，最後 AI 會產出一份完整的 config.json 給你。**

### 3-4：把新的 config.json 貼回 GitHub

1. 複製 AI 產出的 config.json 內容（從 `{` 開頭到最後一個 `}` 結尾）
2. 回到你的 GitHub repo，點 `config.json`
3. 點右上角鉛筆圖示「**Edit**」
4. 全選編輯器裡的內容（Ctrl+A / Cmd+A）
5. 貼上 AI 產出的新內容（Ctrl+V / Cmd+V）
6. 點「**Commit changes**」儲存

> **注意**：記得在 config.json 的 `google_sheet_endpoint` 欄位填入 Step 2 複製的 URL，AI 產出的版本這格通常是空的。

---

## Step 4：開啟 GitHub Pages

1. 進入你的 repo → 上方點「**Settings**」
2. 左側選單點「**Pages**」
3. Source 選「**Deploy from a branch**」
4. Branch 選「**main**」，資料夾選「**/ (root)**」
5. 點「**Save**」
6. 等約 1 分鐘，頁面上會出現你的問卷網址：

```
https://你的GitHub帳號.github.io/你的repo名稱/
```

---

## Step 5：啟用配額控制（若有設定 quota_per_cell）

如果你在 config.json 有設定 `quota_per_cell`（例如 `30`），需要多執行一個步驟，讓系統幫你預先產生隨機分派名單：

1. 回到試算表 → 點「擴充功能」→「Apps Script」
2. 在編輯器上方的函式下拉選單，選「**generateAssignmentQueue**」
3. 點「**執行**」（第一次執行可能會要求授權，請點「允許」）
4. 執行完成後，回到試算表，確認出現「**Queue**」工作表
5. Queue 工作表裡有預先排好的隨機名單，問卷就可以開始發放了

> 如果 `quota_per_cell` 設為 `0`，可以跳過這個步驟，系統會自動改用簡單隨機分派。

---

## Step 6：分享問卷連結

你的 GitHub Pages URL 就是問卷連結，可以直接貼到 LINE / Facebook 發放。

資料會自動進入你在 Step 2 設定的 Google 試算表。
配額模式下，各組人數會依照你設定的份數平均分配。

---

## 之後要更新問卷內容

1. 在 GitHub repo 頁面點 `config.json`
2. 點鉛筆圖示「**Edit**」
3. 修改後點「**Commit changes**」
4. 約 1 分鐘後問卷自動更新

> **注意**：若修改了 `quota_per_cell` 或 factor/level 結構，需要重新執行 `generateAssignmentQueue()` 重建名單。

---

## 常見問題

**問卷開起來空白？**
→ 確認 `config.json` 有成功 commit，且 GitHub Pages 已啟用

**資料沒有進試算表？**
→ 確認 Apps Script 部署時「存取權限」設為「所有人」
→ 回到 Apps Script 編輯器，用 `testWrite()` 函式測試看看

**受測者看到「問卷已達收件上限」？**
→ 這是正常的，代表配額已全部發放完畢

**Queue 工作表沒有出現？**
→ 確認 Step 2 有在程式碼頂部填入 `SURVEY_URL`
→ 確認有執行 `generateAssignmentQueue()`

**AI 產出的 config.json 貼上去問卷壞掉？**
→ 檢查是否只複製了 AI 回覆的一部分（要從 `{` 到最後的 `}` 完整複製）
→ 把錯誤截圖給 AI，請它幫你找問題

**不確定的地方**
→ 先問學長姐
